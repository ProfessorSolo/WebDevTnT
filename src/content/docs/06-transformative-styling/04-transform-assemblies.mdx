# **Order Up!**

Single transforms? Cute.

**Assemblies** are where motion becomes expressive ‚Äî stacking transforms in sequence unlocks complex, cinematic motion (when done right).

## The Golden Rule to Ordered Rule:

> **Transform sequence matters, a lot.**  
> CSS applies transforms **left ‚Üí right**, each one rewriting the coordinate system for the next.

Same transforms.  
Different order.  
Completely different geometry.

## üìò Why order matters ‚Äî in CSS space

<div class="ps-callout">

### üß† **Transforms rewrite space**

Each transform produces a _new coordinate grid_.  
Every transform after that uses the new, warped grid ‚Äî not the original one.

This is why:

- `scaleY(0.5) rotate(45deg)`  
  ‚â†
- `rotate(45deg) scaleY(0.5)`

Transforms are **matrices**, and matrix multiplication is NOT commutative.

Think of transforms like:

- moving and stretching **rubber sheets**
- where each action warps the space the next action happens inside

That's why your rectangle stops looking like a rectangle when you reorder scale + rotate.

</div>
