# Web Image Formats

Welcome to **Format Showdown**, where we pit the most common web image formats
against each other. Each format brings its own strengths, quirks, and ideal use
cases ‚Äî your job is to choose the right tool for the right moment.

This lesson gives you a fast, practical comparison so you always know which
format to reach for during development.

---

## üï∞Ô∏è A (Very) Quick History

Web image formats evolved as the web matured:

- **GIF (1987)** ‚Äî the pixel grandparent: limited colors, looping animations, and the reason PNG needed to exist.
- **JPEG (1992)** ‚Äî lossy, tiny, and perfect for photos.
- **PNG (1996)** ‚Äî lossless, crisp edges, transparency, born from GIF patent drama.
- **SVG (1999)** ‚Äî vector-based, infinitely scalable, essential for UI and icons.
- **WebP (2010)** ‚Äî lightweight modern format supporting lossy, lossless, and transparency.
- **AVIF (2019)** ‚Äî ultra-efficient high-end codec built from AV1 with incredible compression.

This little timeline gives a sense of _why_ we have so many formats ‚Äî and why each still matters.

## üü¶ JPEG ‚Äî The Old Reliable Workhorse

**Best for:** photos, gradients, high-detail scenes  
**Strengths:** tiny file sizes, universal support  
**Weaknesses:** lossy only, no transparency

```md
Use JPEG when:

- you need maximum compatibility
- transparency doesn‚Äôt matter
- the image is photographic or noisy
```

---

## üü© PNG ‚Äî Pixel-Perfect Precision

**Best for:** UI elements, logos, crisp graphics  
**Strengths:** lossless, supports transparency  
**Weaknesses:** can be huge compared to JPEG/WebP

```md
Use PNG when:

- transparency is required
- edges must stay razor sharp
- the image shouldn‚Äôt lose quality
```

---

## üüß WebP ‚Äî Modern Default

**Best for:** almost everything  
**Strengths:** tiny files, lossy + lossless, great quality  
**Weaknesses:** slightly slower to encode

```md
Use WebP when:

- you want smaller files without sacrificing quality
- you need transparency _and_ efficiency
```

---

## üü™ AVIF ‚Äî The High-End Contender

**Best for:** photos, gradients, detailed scenes  
**Strengths:** excellent compression, HDR support  
**Weaknesses:** slower encoding, patchy Safari support

```md
Use AVIF when:

- you want the smallest possible file size at high quality
- you‚Äôre optimizing hero images and background art
```
