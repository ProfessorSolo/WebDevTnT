# **Ready, click, go**

Here we go... motion without a trigger is just decoration.  
**Triggering** is where animation becomes _interactive_, _purposeful_, and _context-aware._

This page covers the essentials:  
Classes ‚Üí Events ‚Üí Scroll ‚Üí Intersection ‚Üí Play/Pause control.

---

## üéõ Triggering with Classes (The Classic)

Still the cleanest approach: toggle a class, fire an animation.

```css
.card {
  animation: fadeUp 500ms ease-out forwards;
  opacity: 0;
}

.card.reveal {
  opacity: 1;
}
```

```js
card.classList.add('reveal');
```

Simple. Direct. No magic.

---

## üñ± Event-Driven Animation (Clicks, Hovers, Presses)

```js
button.addEventListener('click', () => {
  button.classList.add('activate');
});
```

This is perfect for:

- button click feedback
- panel reveals
- navigation transitions
- ‚Äúpress and hold‚Äù moments (paired with direction/play-state)

---

## üåÄ Restarting an Animation

If we need to rerun an animation:

```js
el.classList.remove('run');
void el.offsetWidth; /* Force reflow */
el.classList.add('run');
```

Browser jutsu. Works every time.

---

## üìú Scroll-Triggered Animations

```js
window.addEventListener('scroll', () => {
  const top = el.getBoundingClientRect().top;
  if (top < window.innerHeight * 0.8) {
    el.classList.add('enter');
  }
});
```

Lightweight. Reliable.  
A great bridge to intersection observers.

---

## ‚èØ Play & Pause with animation-play-state

```css
.runner {
  animation: slide 2s linear infinite;
}

.runner.paused {
  animation-play-state: paused;
}
```

```js
runner.classList.add('paused');
runner.classList.remove('paused');
```

Perfect for:

- scroll-controlled sequences
- hold-to-activate behaviors
- drag interactions
- toggled looping animations
