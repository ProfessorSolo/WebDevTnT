# **Take it Eeeeeasy - Time Bender**

Same distance. Same duration.  
RADICALLY different vibes.

Welcome to the personality layer of motion.

Timing functions are how we tell the browser _how_ to move through time ‚Äî chill, snappy, floaty, mechanical, or unhinged.

It's all about the curves, and wrinkles, of the timeline.

---

## üéö The Easing Lineup

### **ease**

Your default. Smooth start, smooth end.

```css
transition-timing-function: ease;
```

### **linear**

No acceleration. No deceleration.  
Just‚Ä¶ math.

```css
transition-timing-function: linear;
```

### **ease-in**

Slow at first, then rockets forward.

```css
transition-timing-function: ease-in;
```

### **ease-out**

Fast start, gentle finish.  
Perfect for hover exits or UI elements that ‚Äúland softly.‚Äù

```css
transition-timing-function: ease-out;
```

### **ease-in-out**

Slow ‚Üí fast ‚Üí slow.  
The diplomat ‚Äî great for general UI motion.

```css
transition-timing-function: ease-in-out;
```

---

## üß™ Custom Curves ¬∑ `cubic-bezier()`

Want personality? This is where we craft it.

```css
transition-timing-function: cubic-bezier(0.68, -0.55, 0.27, 1.55);
```

That one overshoots _hard_.  
Try a few more:

```css
/* Smooth */
cubic-bezier(0.25, 0.1, 0.25, 1);

/* Snappy */
cubic-bezier(0.4, 0, 0.2, 1);

/* Dramatic overshoot */
cubic-bezier(0.7, -0.5, 0.3, 1.4);
```

Every curve tells a story.

## üìò How timing functions work

<div class="ps-callout">

### üß† **These curves don't show position ‚Äî they show time**

The graph isn‚Äôt the bunny‚Äôs path.  
It‚Äôs the **pace** of the motion ‚Äî how quickly or slowly the animation moves through time.

That‚Äôs why:

- A steep curve = fast moment
- A flat curve = slow moment
- Ease-in starts slow
- Ease-out ends slow
- Ease-in-out slows at both ends
- Custom cubic-beziers can overshoot, bounce, or snap

The **transform** defines _where_ the bunny goes.  
The **timing function** defines _how it feels getting there_.
