## üü• First-Born Highlight ‚Äî `:first-child` and Friends

We wanted the **first story row** to feel special.

```css
/* Option A ‚Äî first child of the story list */
.story-list > .story-row:first-child {
  /* red border & glow */
}

/* Option B ‚Äî equivalent with :nth-child() */
.story-list > .story-row:nth-child(1) {
  /* red border & glow */
}
```

Both read as ‚Äúthe first child of `.story-list` that is a `.story-row`.‚Äù

- `:first-child` is a little easier to scan.  
- `:nth-child(1)` becomes handy when we later refactor to `2n + 1` patterns and want the consistency.

---

## üüß Rhythm Banding ‚Äî `:nth-child()` vs `:nth-of-type()`

We also wanted a repeating orange band on every other **story row**.

Because the `div.interlude-row` lives between rows 2 and 3, we have a choice:

```css
/* Option A ‚Äî band every odd child, counting the interlude */
.story-list > *:nth-child(odd) {
  /* background band */
}

/* Option B ‚Äî band every odd story-row only, ignoring the interlude */
.story-list > .story-row:nth-of-type(odd) {
  /* background band */
}
```

- Option A says, ‚ÄúEvery odd child, whatever it is.‚Äù  
- Option B says, ‚ÄúEvery odd `.story-row`, ignoring other types in the list.‚Äù

In the minimo, we used **type-based banding** so that the ad doesn‚Äôt break the visual rhythm across stories.

---

## üíó Type-Based Marker ‚Äî `:nth-of-type()` in Action

We wanted a pink badge on the **third story**, even though it‚Äôs not the third child (the ad interrupts).

```css
/* Third story-row of its type */
.story-list > .story-row:nth-of-type(3) .story-kicker::before {
  content: "TYPE¬∑III";
  /* pink badge styles */
}
```

If we had used `:nth-child(3)`, the selector would have grabbed the `interlude-row` instead, because it is the actual third child.

`nth-of-type()` lets us say:

> ‚ÄúThird `div.story-row` among its siblings, ignoring other tags.‚Äù

---

## üßµ Soft Landing ‚Äî `:last-child` vs `:last-of-type()`

Finally, we may want to soften the last story row‚Äîremoving extra borders or adding spacing.

```css
/* Option A ‚Äî last child of the list, whatever it is */
.story-list > .story-row:last-child {
  /* soften bottom border, margin, etc. */
}

/* Option B ‚Äî last story-row, even if something comes after it */
.story-list > .story-row:last-of-type {
  /* soften bottom border, margin, etc. */
}
```

Right now, both options behave the same because the last child happens to be a `.story-row`.

If we later added a footer or another `div` beneath the stories:

- `:last-child` would stop matching the final story.  
- `:last-of-type` would continue to treat the final story-row as ‚Äúlast.‚Äù

---

## üß† Takeaways

Relational selectors let us:

- encode **position** directly into the selector,  
- choose whether to count **all children** or only those of a certain **type**,  
- avoid adding extra ‚Äúfirst/last/featured‚Äù classes in simple cases.

When deciding between them, we can ask:

- ‚ÄúIf I insert an extra element in the middle, will this selector still mean what I think it means?‚Äù  
- ‚ÄúAm I really counting children, or am I counting *stories*, *rows*, or some other conceptual group?‚Äù

When the answer is clear, the rune almost picks itself.
